fgets and sys.stdin and readline all blocks waiting for input until there is something to read from stdin/fgets. then reeads one line and returns it. So the harness will print "ready" and then wait for the next line. The next line is the signal to start the timer and run the command. After the command is done, the harness will print the checksum and stop the timer. This way we can measure the time taken by the command to run and also verify the output using checksum.

\subsubsection{Implementation-specific design decisions}
Each language's implementation follows the stdin/stdout handshake protocol. The program prints \texttt{"ready"} on startup, then blocks waiting for a newline trigger from the measurement harness before executing the algorithm. The specific I/O primitives used, and the rationale for each choice, are described below: 


\paragraph{C -}
    fgets

\paragraph{Python -}
    stdin/stdout

\paragraph{JavaScript -}
    readline
\paragraph{Java -}
    public static void main \\
    buffered reader/writer

\paragraph{Go -}
    bufio

\paragraph{Rust -}
